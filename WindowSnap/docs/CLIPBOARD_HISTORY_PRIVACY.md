# Clipboard history privacy and migration

WindowSnap treats clipboard history as sensitive local data. It does not upload
history, add analytics, or associate history with an identity.

## Storage

Persistent history is stored at:

`~/Library/Application Support/WindowSnap/clipboard-history.json`

Writes use atomic replacement and the history and recovery files use POSIX mode
`0600` (readable and writable only by the current user). The containing
`WindowSnap` directory is created with mode `0700`.

The current valid snapshot is also atomically written to
`clipboard-history.backup.json`, so deleted or expired entries are not retained
in a stale backup. If the primary file is corrupt or partially written,
WindowSnap loads the backup and atomically restores it. If neither file
can be decoded, WindowSnap starts with an empty history without crashing. Valid
entries in a partly corrupt JSON array are preserved.

## Migration from UserDefaults

Older versions stored JSON under the `ClipboardHistory` UserDefaults key. On the
first launch after upgrading, WindowSnap:

1. decodes the legacy payload;
2. atomically writes it to Application Support;
3. applies owner-only permissions; and
4. removes the UserDefaults payload only after the file write succeeds.

If migration cannot write the new file, the legacy payload remains in place so
the migration can be retried. If both a new file and a legacy payload exist, the
new file is authoritative and the stale legacy payload is removed, preventing
duplicate entries.

## Capture and retention rules

- Pasteboards marked `org.nspasteboard.TransientType`,
  `org.nspasteboard.ConcealedType`, or `org.nspasteboard.AutoGeneratedType` are
  never recorded.
- Sensitive-data pattern filtering is always enabled and has no off switch. It
  is defense in depth; privacy markers remain the primary capture boundary.
- Retention choices are session only, 1 day, 7 days (default), and 30 days.
- Time-based cleanup removes expired unpinned entries. Pinned entries remain
  until unpinned, deleted, or removed with **Clear All History**.
- Session-only history remains in memory until WindowSnap quits and is never
  written to disk.
- Pausing monitoring is persistent and prevents new captures until resumed.
- **Clear All History** removes in-memory history, the primary file, the backup,
  and any remaining legacy payload.

Production log messages may include an item type and byte count, but never
clipboard text, URLs, rich text, image data, thumbnails, or previews.
